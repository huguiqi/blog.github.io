读一本禁书的笔记

从2015年过期间到2016年3月底，这本书我终于看完了，全世界的国家都把它列为禁书，他的书会让人发狂，可以让整个民族亢奋，也可以毁灭世界。。。。。

我曾经在youtube上和一位台湾网友争论，什么是法西斯政权，什么是纳粹？很遗憾，台湾的这一代年轻人太弱了，不但对大陆这边的人有非常深的偏见(不停的咒骂)，而且对世界的了解出乎意料的浅薄，这让我非常失望，想当年从大陆去了台湾的人可是中国精英中的精英啊，可惜了，21世纪这一代台湾人，真的让我大跌眼镜。。。。。

这位网友认为：

  1. 大陆人民都是被洗脑了的，生活、文化水平没办法与台湾比
  2. 大陆是非常黑暗的，这位网友非常不客气的叫中国人是支那人(这是我最愤怒的)
  3. 他认为共产主义就是纳粹，希特勒当年实行的就是社会主义制度(读过历史的都知道，纳粹是最仇视社会主义、共产主义的)
  4. 他认为纳粹的意思就是社会主义(但他不知道，什么是民族社会主义,就多了个民族，却是完全不一样的东西)
  5. 法西斯就是当年德国(但其实法西斯是意大利墨索里尼政权)
  6. 台湾是民主国家，制度比大陆优越，大陆马上要完蛋了(这种优越感真不知道从哪里来的)

什么是 Docker

Docker 是一个开源项目，诞生于 2013 年初，最初是 dotCloud
公司内部的一个业余项目。它基于 Google 公司推出的 Go 语言实现。
项目后来加入了 Linux 基金会，遵从了 Apache 2.0 协议，项目代码在
GitHub 上进行维护。

Docker 自开源后受到广泛的关注和讨论，以至于 dotCloud 公司后来都改名为
Docker Inc。Redhat 已经在其 RHEL6.5 中集中支持 Docker；Google 也在其
PaaS 产品中广泛应用。

Nginx 性能优化

Nginx作为一个非常流行和成熟的Web Server和Reserve Proxy
Server，网上有大量的性能优化教程，但是不同的业务场景千差万别，什么配置是最适合自己的，需要大量的测试和实践以及不断的优化改进。最近用户调用量突破百万大关之后，就遇到了一些问题，虽然不算太复杂，但也折腾了挺长时间才搞定，积累了不少经验。

碰到的这个问题其实已经有一段时间了，有客户给我们反馈调用超时，但是我们自己从系统监控上看都是正常的，只有几十毫秒肯定不会超时，怀疑是不是网络的原因，但是出现几次后，就隐隐感觉这个问题可能不是偶发性的，应该还有深层次的原因。

因为我们服务面向企业客户的，虽然每家客户的调用量可能会非常大，但每家企业客户就那么几个公网IP，即使以后有上千家客户，Nginx也可以轻松支撑这些并发连接。因此，首先先从网络上对Nginx长连接作了优化，将长连接从原来配置的5秒钟改成5分钟，将每次建立连接请求的数目从默认的100调整到1000。

调整完毕后，通过netstat
-anp命令可以看到，新建连接请求会减少，说明长连接已起到作用。但过了一段时间，仍然发现有客户调用超时的情况发生，从Nginx日志中可以看到请求时间还是有超过1s的，甚至有长达20s左右的，如下所示：
