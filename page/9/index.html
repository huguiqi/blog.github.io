创建JSP自定义标签



### 创建JSP自定义标签



\



\

在JSP编程中，Struts一般用来处理form和业务逻辑，Struts中使用了很多自定义标签，如html、bean等，但很多时候我们还是需要使用

“<% java code %>”
这样的语句来操作一些输出。为了减少这种代码，可以创建自定义标签。



\



一个tag就是一个普通的java类，它唯一特别之处是它必须继承TagSupport或者BodyTagSupport类。这两个类提供了一些方法，负责jsp页面和你编写的类之间的交互，例如输入，输出。而这两个类是由jsp容器提供的，无须开发人员自己实现。换句话说，你只需把实现了业务逻辑的类继承TagSupport或者BodyTagSupport，再做一些特别的工作，你的类就是一个Tag。并且它自己负责和jsp页面的交互，不用你多操心。

二十多岁是要安逸还是要拼搏

By sam

发表于 2018-11-19

领悟

4月份在一家传统型企业为核心的公司上班，上海分部原来设立时，主要是地面销售人员，当时面试时，公司老板直接从东莞飞到上海，为了是让我建立自己的一支软件技术团队，以便打造公司未来的产品人机一体化生态产业链。
当时听到这里时，心里是一阵悸动，决定和公司一起把产品做成我认为的样子，使命感十足。

团队在陆续的招聘中，我需要的人员配备基本完成。需求也在不断的与杭州的产品经理不断确定与升级中，虽然他们的产品设计的非常不合理，产品文档让人摸不着头脑，但本着合作的精神，只能一直鼓励与影响着他们持续改进，并期待将来产品文档会越来越详细并有条理。

需求—技术—硬件分割在三地，一直以来都是我最担心的，这方面最大的问题就是沟通不到位，鸡同鸭讲，但我认为如果保持高度沟通与高度互信，这个问题可以逐步减少，也许可以忽略这个问题。
但事实证明，这方面的成本是不可能被忽略的，所占的比重只会随着产品的发布，将来会有越来越多的隐患，如果可以，我真的非常建议软件技术团队和硬件技术团队合并，统一放到一个地方，这样至少技术这块可以达成一致，而且沟通起来可以非常方便。

这边的项目，目前招的人，如果按照我招人的标准，这些人其实都不合格，都比较初级，无奈，公司老板虽然说对招人薪资标准没有限制，但其实还是有限制的，因为老板不好意思明说，只好授意给HR，给的标准范围都是10-14k，但这个区间的人，目前我看都是在11k左右的标准招的人，在上海，2016年这样，我面试的人都只能说，可以做事，但无法作为技术骨干，只能培养，无法解决很多难题，他们有一个共同的弱点，就是没有解决问题的方法。对问题的解决能力需要很多项目去锻炼，去磨合。

能力上和思维上，有同样的心态的人，对于团队来说，太重要了，90后这一代人和80后这一代人的生活方式真的不一样，他们很多上班，要的是安逸的工作环境与生活，而不是拼博，对我来说，这么年轻，还要这么安逸的生活，不上进，不拼搏，简直就是人生的罪恶。安逸是50岁以上人的专属，20多岁的人，需要安逸？？？？我的天。。。。。。我的团队有个90后的小朋友，他说经常加班会把身体加坏的；可是我们并不是996啊，公司需要业务活动的支撑，就需要一定强度技术支持呀，其实我们离996还差一点，如果说，加班是因为管理不善，那大公司会加班吗?我总是跟他们说；其实我们已经很幸福了;但是得到的回应却是一脸的不信。

linux下安装软件不像windows一样点击鼠标就可以完成，然而之前接触到的是从软件仓库里使用yum或者apt的工具来安装。最近接触到的服务器版本比较老，没有yum之类的软件，安装软件都是从源码安装。由于安全限制，服务器还不能直接访问外网（甚至两个子网都不能互通），wget方式获取软件源码的方法都不可行。由于路由的隔离，只能在能访问外网的机器上下载好软件，然后使用rz或者samba的方式拷贝到测试机器上，然后开发机使用ftp的方式从测试机上获取到安装包（scp和ssh都被屏蔽掉）。当然，这是安装前的工作。

Disable Network Manager

  1. First stop and disable the gnome network manager from running on boot.

    systemctl stop NetworkManagerservice

systemctl disable NetworkManagerservice

  2. Now start and enable the network service to run on boot.

    systemctl restart networkservice

systemctl enable networkservice

Set the Static Address

  1. Check which interface(s) you want to set as static.
  1. Next edit the config file for the required interface.

sysconfignetworkscriptsifcfg

  1. Edit the config file to look similar to the following.

Change the BOOTPROTO from “dhcp” to “static”. Also adding IPADDR, NETMASK, BROADCAST and NETWORK variables and make sure ONBOOT is set to yes.

"e88f1292-1f87-4576-97aa-bb8b2be34bd3"
NM_CONTROLLED"yes"
HWADDR"D8:D3:85:AE:DD:4C"

DEVICE"em1"
ONBOOT"yes"

GATEWAY192.168

Apply the Settings

Restart the network service to apply the settings.
