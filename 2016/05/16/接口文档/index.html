    1.1.1 符号定义

    序号 符号缩写 符号性质 符号说明
    1 M 强制域(Mandatory) 此域必选，否则将被认为格式出错
    2 C 条件域(Conditional) 此域在一定条件下必选
    3 O 选用域(Optional) 此域由发送方自选

    1.1.2 全局变量定义

    1.2. 协议

    • 目前仅提供HTTP通信协议采用RESTful风格；
      • 请求提供GET,POST两种方式；
      • 接受的请求参数JSON或者表单形式
      • 返回结果均采用JSON；

    注意

    1. 请求参数为JSON模式，请求头需设置Content-Type: application/json;
    2. 请求参数表单模式须设置请求头为Content-Type:application/x-www-form-urlencoded;
    3. 请求头需要增加一个KEY名为 G-Agent 值组成为：app名称简写;app版本号;设备类型;设备系统版本 设备类型iphone为1，安卓为2
    4. 判断是否登录的标识符 如果登录成功后 须在请求头（或者放在queryString参数上）设上 userId 值
    5. 签名算法为 G-Agent + 接入KEY(token) + userId(userId 未登录传空字符串) 组合字符串的MD5值， 并在请求头设（或者放在queryString参数）上sign值

    签名验证

ticket=""
token="3E8802D881C51$%DE36FE0C#CA6C4F4D,233E8802D881C519DE36FE0CCA6C4F4D23"
sign = MD5(G-Agent + token + userId) = "d7251d35ea1ec87bda9794e8b0e3219b"

    1.3 编码

    所有接口采用 UTF-8

    1.4 公共参数

    所有的请求都有以下参数，以便识别请求来源是否合法

    • request:
    参数名称 是否必须 描述 用途
    sign 是 签名 md5(uid + token)
    token 是 令牌key 分发给访问渠道的令牌
    uid 是 用户ID，默认是0 唯一标识用户
    p 否 页码 分页查询使用
    r 否 每页显示条数 分页查询使用
    channel 是 客户端访问渠道:ios,android,wechat,mobileWeb 区分访问来源
    version 否 版本号 只有app有值

    1.4.2 响应码定义

    序号 code值 符号说明
    1 200 处理成功
    2 400 参数输入不正确
    3 401 session失效需要重新登录
    4 404 资源链接不存在，或者请求的数据资源不存在
    5 406 签名不正确
    6 500 业务处理失败
    7 10000 10000以上表示自定义错误,如验证手机号码

    用户接口

    ## 2.1 验证手机号码是否被注册

    请求：

    GET http://${domain}:${port}/user/mobile

    输入

    {
 "mobile": "13524101981" 11位手机号
}

    输出

    {
"code": 10001,
"msg": "手机号码已存在"
}

    注意：只有code为200的情况下，手机号可用

    2.2 用户注册

    请求：

    POST http://${domain}:${port}/user/register

    输入

    {
 "mobile": "13524101981", String、11位手机号
 "password":"123456", String、用户密码
 "repassword":"123456", String、确认密码
    "verificationCode":"4532",String 验证码
}

    输出

    {
"code": 200,
"msg": "success",
"data":{
"uid":"8686ae2da1c217df4815b90a8a6c2abf",
"mobile":"13524101981"
}    
}
    code msg
    10008 验证码失效了
    10007 验证码不正确
    10003 手机号已经存在

    注意：只有code为200的情况下，注册成功并返回uid、手机号码

"msg": "success",

    设备信息

    4.1 设备信息列表API

    请求：

    GET http://${domain}:${port}/device/findById

    输入：

    "uid": "1" String、用户ID

    输出：

    {
    "code":200,
    "msg":"success",
    "data":[
                {
                    "name":"anne",
                    "did":"1"
                }
            ]
}

    注意：当result为空数组时，该用户下没有设备

    4.2 设备信息添加API

    请求：

    POST http://${domain}:${port}/device/save

    输入：

    {
    "did":"1",   String、唯一ID
    "name":"anne",   String、设备名称
    "type": 1,      Int、设备类型  1.消毒底座  2.牙刷  3.手环)
    "uid": "1",  String、用户ID
    "bluetoothName": "lbd" String、蓝牙名称
}

    输出：

    {
    "code":200,
    "msg":"success"
}

    注意：当code为200的情况下，设备信息添加成功


    广告列表接口

    请求：

    POST http://${domain}:${port}/ad/list

    参数:

    默认参数

    输出：

    {
    "code":200,
    "msg":"success",
    "data":[
                {
                    "adId":1,
                    "adType":1,
                    "imgUrl":"http://xxxx/xxx.png",
                    "title":"广告一",
                    "flag":0
                },
                {
                    "adId":2,
                    "adType":2,
                    "imgUrl":"http://xxxx/xxx.png",
                    "title":"广告一",
                    "flag":2
                },
                {
                    "adId":3,
                    "adType":3,
                    "imgUrl":"http://xxxx/xxx.png",
                    "title":"广告一",
                    "flag":3,
                    "appPage":1001
                }
            ]
}

    返回参数说明：

    参数名称 描述 用途
    adId 广告ID 广告唯一标识
    adType 广告内容类型：1、2、3，分别代表:url、content、appPage app跳转,如果是url则跳转到app内部网页，content则跳到原生固定详情页面，appPage则跳转到app指定页面
    appPage 跳转到某个页面的名称 adType为3的情况下才会有值，用来跳转到app指定页面，内容为app页面码
    imgUrl 广告图片地址 显示在手机banner页面上
    title 广告标题 广告标题
    flag 0（普通）、1(特价促销)、2(火爆) 、3(秒杀) 在手机上banner上显示特殊标记
