二十多岁是要安逸还是要拼搏

By sam

发表于 2017-02-08

领悟

4月份在一家传统型企业为核心的公司上班，上海分部原来设立时，主要是地面销售人员，当时面试时，公司老板直接从东莞飞到上海，为了是让我建立自己的一支软件技术团队，以便打造公司未来的产品人机一体化生态产业链。
当时听到这里时，心里是一阵悸动，决定和公司一起把产品做成我认为的样子，使命感十足。

团队在陆续的招聘中，我需要的人员配备基本完成。需求也在不断的与杭州的产品经理不断确定与升级中，虽然他们的产品设计的非常不合理，产品文档让人摸不着头脑，但本着合作的精神，只能一直鼓励与影响着他们持续改进，并期待将来产品文档会越来越详细并有条理。

需求—技术—硬件分割在三地，一直以来都是我最担心的，这方面最大的问题就是沟通不到位，鸡同鸭讲，但我认为如果保持高度沟通与高度互信，这个问题可以逐步减少，也许可以忽略这个问题。
但事实证明，这方面的成本是不可能被忽略的，所占的比重只会随着产品的发布，将来会有越来越多的隐患，如果可以，我真的非常建议软件技术团队和硬件技术团队合并，统一放到一个地方，这样至少技术这块可以达成一致，而且沟通起来可以非常方便。

这边的项目，目前招的人，如果按照我招人的标准，这些人其实都不合格，都比较初级，无奈，公司老板虽然说对招人薪资标准没有限制，但其实还是有限制的，因为老板不好意思明说，只好授意给HR，给的标准范围都是10-14k，但这个区间的人，目前我看都是在11k左右的标准招的人，在上海，2016年这样，我面试的人都只能说，可以做事，但无法作为技术骨干，只能培养，无法解决很多难题，他们有一个共同的弱点，就是没有解决问题的方法。对问题的解决能力需要很多项目去锻炼，去磨合。

能力上和思维上，有同样的心态的人，对于团队来说，太重要了，90后这一代人和80后这一代人的生活方式真的不一样，他们很多上班，要的是安逸的工作环境与生活，而不是拼博，对我来说，这么年轻，还要这么安逸的生活，不上进，不拼搏，简直就是人生的罪恶。安逸是50岁以上人的专属，20多岁的人，需要安逸？？？？我的天。。。。。。我的团队有个90后的小朋友，他说经常加班会把身体加坏的；可是我们并不是996啊，公司需要业务活动的支撑，就需要一定强度技术支持呀，其实我们离996还差一点，如果说，加班是因为管理不善，那大公司会加班吗?我总是跟他们说；其实我们已经很幸福了;但是得到的回应却是一脸的不信。

传统行业公司涉足互联网软件开发之我见

2016年过去了，还有三天就要回老家过年了；这一年走来，总结下来，有喜有悲，有聚有散；
生活上，终于找到了我的另一半，因为她，今年我再累，心也是甜的。今年我将会带她见我的父母，如果顺利的话，我们将在17年国庆举行婚礼。

我的公司老板，有一颗做国产第一支智能电动牙刷的心，但是从第一天见他时，从他身上就透露着一身江湖草莽义气，当时觉得，讲义气的人，人品不会太差，老板的话更不会，再加上，我也觉得挻有意思，也想做做这个方面的东西，所以就答应建立团队，给他做出个像样的东西。。。。。
但一年过去了，双十一的app版本，说句心里话，我自己都非常不满意的；
首先，app的使用并不流畅，很多地方用起来非常容易搞错，比如说：二维码扫描的入口，智能非智能用户完全搞不清楚；经常出现很多用户联系投诉；
但是，从这一年与杭州的沟通交流中种种的不畅，让我彻底意识到，出现这些问题，并不能把原因简单的归咎于运营/产品、软件、硬件分属三地，导致沟通不畅；
我总结下来主要有这几点：

  • 运营与产品信息更新不及时； 产品对技术信息曝光不及时；
  • 硬件方面问题全部都体现在软件上，用户只能去怪软件，差评只会说软件的问题，因为入口就是软件
  • 不专业团队指挥技术团队；
  • 技术团队初期招人不慎，导致代码超难维护；
  • 产品原型的缺失与需求不清晰导致设计漏洞太多;

阿里云服务器环境配置

By sam

发表于 2016-09-18

阿里云服务器环境配置

数据盘初始化

如果在购买ecs服务器时，选择购买了数据盘，则在linux系统登陆进去，执行命令：

fdisk -l

查看挂载的所有数据盘，注意：

df -hT    //无效，只能看到已经初始化的挂载盘

运行 fdisk -l 命令查看数据盘。
注意：在没有分区和格式化数据盘之前，使用 df -h 命令是无法看到数据盘的。
在下面的示例中，有一个 5 GB 的数据盘需要挂载。
如果执行了 fdisk -l 命令后，没有发现 /dev/xvdb ,则不需要格式化。

  • 运行 fdisk /dev/xvdb，对数据盘进行分区。

根据提示，依次输入 n，p，1，两次回车，wq，分区就开始了。

  • 运行 fdisk -l 命令，查看新的分区。

新分区 xvdb1 已经创建好。如下面示例中的/dev/xvdb1。

  • 运行 mkfs.ext3 /dev/xvdb1，对新分区进行格式化。

格式化所需时间取决于数据盘大小。您也可自主决定选用其他文件格式，如 ext14 等。

  • 运行 echo /dev/xvdb1 /mnt ext3 defaults 0 0 >> /etc/fstab 写入新分区信息。

完成后，可以使用 cat /etc/fstab 命令查看。

  • 运行 mount /dev/xvdb1 /mnt 挂载新分区，然后执行 df -h 查看分区。

如果出现数据盘信息，说明挂载成功，可以使用新分区了。

 mount /dev/xvdb1 /mnt
df -h
 Filesystem      Size  Used Avail Use% Mounted on
 /dev/xvda1       40G  1.5G   36G   4% /
 tmpfs           498M     0  498M   0% /dev/shm
 /dev/xvdb1      5.0G  139M  4.6G   3% /mnt
